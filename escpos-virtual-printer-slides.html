Android ESC/POS Virtual Printer

TCP로 받은 프린터 명령을 화면의 영수증으로 재구성한다

핵심은 소켓 서버, 바이트 스트림 파서, 영수증 문서 모델, 스크롤 가능한 UI를 서로 분리하는 것이다.

Product Shape

앱은 프린터처럼 열리고, 뷰어처럼 보여준다

기본 포트는 9100. POS가 보낸 ESC/POS 바이트를 수신하고, 컷 명령 또는 timeout으로 영수증을 완료한다.

Pipeline

처리 흐름은 다섯 단계로 고정한다

TCP Serveraccept loop, client session, byte chunk
Stream Bufferpartial command, partial multibyte text
ESC/POS ParserESC, GS, FS, DLE command state machine
Receipt Builderline, style, alignment, image, cut event
Receipt UIlist, detail, scroll, debug log
Parser Decision

파서는 상태 머신이어야 한다

TCP는 경계가 없다 명령어가 chunk 중간에서 끊긴다. ESC a n의 n이 다음 read에 올 수 있다.
텍스트도 끊긴다 UTF-8 또는 CP949 같은 멀티바이트 문자는 바이트 일부만 먼저 도착할 수 있다.
명령이 다양하다 Epson 명령 목록은 문자, 위치, 컷, 이미지, 바코드, 2D 코드 등으로 넓다. 알 수 없는 명령은 경고로 복구한다.
MVP Command Set

1차 버전은 텍스트 영수증을 정확히 표시하는 데 집중한다

명령역할앱 처리
ESC @초기화스타일, 정렬, 코드페이지 기본값 복원
LF, CR줄 처리현재 라인을 확정하거나 옵션에 따라 무시
ESC a n정렬left, center, right 상태 변경
ESC E n, ESC - n굵게/밑줄TextSegment 스타일에 반영
GS ! n문자 크기폭/높이 배율에 매핑
GS V용지 컷현재 영수증 완료 이벤트
Native Android

네이티브 구현은 서버 앱 성격에 가장 잘 맞는다

권장 스택
  • Kotlin, Coroutines, Flow
  • Foreground Service
  • Jetpack Compose
  • Room 또는 DataStore
  • 순수 Kotlin ESC/POS parser
장점
  • TCP 수신과 백그라운드 유지가 안정적
  • 파서 단위 테스트가 쉽다
  • Bridge 비용이 없다
  • 장시간 켜두는 POS 보조 앱에 적합
React Native

React Native는 UI 생산성이 장점이지만, 서버 코어는 Native에 둔다

JS 영역 영수증 목록, 상세 화면, 설정, 검색/필터, 상태 관리
Native 영역 TCP 서버, Foreground Service, ESC/POS 파서, 영수증 완료 감지
Module API start(port), stop(), getStatus(), receiptCompleted 이벤트

Risks

초기부터 막아야 할 리스크

부분 명령chunk 중간 분리를 테스트 케이스로 고정한다.
인코딩기본 UTF-8 외에 CP949/EUC-KR 옵션을 둔다.
제조사 확장UnknownCommand를 보존하고 원본 바이트를 추적한다.
Android 정책사용자 조작으로 서버 시작, FGS type/권한을 명시한다.
Recommendation

최종 권장안: Kotlin Native 우선

1차 제품은 Kotlin + Foreground Service + Jetpack Compose로 구현한다. React Native를 선택하더라도 TCP 서버와 ESC/POS 파서는 Android Native Module로 유지한다.
